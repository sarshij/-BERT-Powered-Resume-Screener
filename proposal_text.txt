vi\n1. INTRODUCTION
1.1 Background
Hiring the right people is key for any company because it affects how well the business
runs and grows. Usually, HR staff have to go through hundreds or even thousands of
resumes for one job. They manually check if each candidate has the right skills and
experience. This takes a lot of time and effort, and it’s easy for people to make mistakes
or be biased when they have so much to read [1].
With more people using job portals online, the number of applications has gone up
a lot. Research shows that a single job posting can get around 250 resumes, and about
75 to 88% of them usually aren’t right for the role [2]. This makes it very hard for HR
teams, especially in smaller companies that don’t have many recruiters.
To help with this, automated screening systems were created. At first, they just
looked for specific keywords using methods like TF-IDF and cosine similarity [3].
Lately, things have improved with Natural Language Processing (NLP) and models
like BERT [4]. A version called Sentence-BERT (SBERT) can turn text into numerical
vectors that capture the actual context [5]. This lets the system compare resumes
based on meaning rather than just matching words, which makes the results much more
accurate.
At the same time, AI writing tools have made it easy for people to create fake or
“inflated” resumes. These resumes might look good to a simple keyword scanner, but
they don’t always show the candidate’s real skills [6]. This means we need systems that
don’t just find matches but also check if the information is actually true and consistent.
1.2 Problem Statement
Despite new tech in hiring, there are still major problems with how resumes are screened
today. If someone uses different words for the same skill, they might get rejected by
mistake. This problem makes it hard for the system to understand that a candidate
with “Deep Learning” experience is also a match for “Neural Networks.” Furthermore,
resumes come in different layouts and formats like PDF or DOC, and extracting clean,
structured text from them is technically difficult. Current systems often fail to parse this
data accurately, leading to missing information and poor matching.
Another growing concern is that many systems don’t check if a resume is actually
true. They focus only on ranking candidates by how well they match the job description,
but they can’t see flags like fake job dates or graduation years that don’t align with
experience. With the rise of AI tools, many people are now using AI to write “perfect”
resumes that might not even be real. These AI-made resumes look great to a simple
1\nkeyword scanner but often hide the fact that the candidate doesn’t have the actual skills
needed. This makes it much harder for HR teams to trust the candidates they are seeing.
Human bias is also still a problem. Even when using digital tools, recruiters might
still be influenced by things like formatting or the names of schools. This leads to
unfair results. On top of that, manual screening just doesn’t scale. When a single job
gets hundreds of applications, it’s almost impossible for a human to read them all in a
reasonable time. This leads to long hiring delays and can cause companies to lose great
candidates to competitors. There is a clear need for a better system that doesn’t just
match keywords but actually understands the meaning and verifies the authenticity of
every resume.
1.3 Objectives
•To develop a semantic resume-job matching system using Sentence-BERT (SBERT)
for accurate similarity analysis.
•To design a hybrid scoring model integrating semantic similarity, skill overlap,
and experience relevance weighting.
•To implement a resume authenticity validation module using a Decision Tree
classifier to detect inconsistencies.
•To build a recruiter dashboard for candidate ranking, profile analysis, and system
performance evaluation.
1.4 Scope of the Project
The project focuses on the design and development of a resume screening and authentic-
ity validation system for automated candidate evaluation. The system provides semantic
matching and structured validation using NLP and machine learning techniques.
•Target Users:HR professionals, recruiters, and small-to-medium enterprises
requiring efficient resume screening.
•Input Support:
•Resume upload in PDF and DOC formats.
•Job description input via text field or file upload.
•Core Capabilities:
•Semantic resume-job matching using SBERT embeddings.
•Custom weighted scoring with configurable parameters.
•Resume authenticity validation using rule-based feature extraction.
2\n•Decision Tree classification for genuine vs. suspicious resume detection.
•Candidate ranking with detailed analysis reports.
•Operational Boundaries:The system is intended for academic research and
prototype demonstration. It does not integrate with external databases for real-
time certificate or employment verification.
3\n2. LITERATURE REVIEW
2.1 Related Work
Resume screening has come a long way in the last ten years, thanks to better AI and
language processing. In the past, systems mostly just looked for keywords, but now they
use deep learning to actually understand the text.
At first, screening tools used keyword matching and simple parsers to find info
in resumes [3]. These were fast but they couldn’t understand how different words
relate. For example, a candidate might mention “Deep Learning” while the job asks for
“Neural Networks.” Even though these are closely related, old systems would mark it
as a mismatch.
Later, methods like TF-IDF and cosine similarity were used to see which words were
more important [7]. But even then, these tools didn’t really get the context or the order
of words.
Then came word embedding techniques like Word2Vec [8] and GloVe, which helped
machines see relationships between words. Doc2Vec even tried to do this for whole
documents [9]. These were better than just keyword matching, but they still struggled
with complex resumes or long job descriptions.
Huge progress was made with transformer models like BERT [4]. A version called
Sentence-BERT (SBERT) [5] was designed specifically to compare sentences and pas-
sages efficiently. This is perfect for comparing a candidate’s profile with a job description
because it focuses on the overall meaning.
Recent studies show that these embedding-based methods work much better than
old keyword-based ones when it comes to finding the right person for a job [1][2].
2.1.1 Existing Systems
Several commercial and academic systems have been developed for automated resume
screening:
•Ideal:An AI-powered resume screening tool integrated with Applicant Tracking
Systems (ATS) that uses machine learning to evaluate and rank candidates [10].
•Freshworks AI:Provides contextual resume screening beyond simple keyword
matching, incorporating candidate ranking and assessment features.
•HireVue:Utilizes AI-driven candidate evaluation including resume analysis, with
some systems also incorporating video interview analysis.
4\nWhile these systems offer varying degrees of AI-based screening, most focus pri-
marily on candidate ranking and do not explicitly address resume authenticity validation
or AI-generated resume detection.
2.1.2 Research Gaps
Even with all this progress, there are still some things missing in current screening tools:
•Most tools only care about matching skills but don’t check if the resume itself is
authentic.
•Very few systems combine good matching with a way to detect fraud or inconsis-
tencies.
•Detecting resumes made by AI tools is still a new area and isn’t usually part of
these screening systems.
•There isn’t much research on hybrid scoring models that look at semantic meaning,
actual skills, and years of experience all at once.
•Small companies need affordable solutions that aren’t too complex to set up.
Our project aims to fill these gaps by combining SBERT matching with a validation
engine that uses a Decision Tree classifier.
2.2 Related Theory
This chapter presents the theoretical foundations underlying the proposed AI-based

from natural language processing, semantic text representation, information retrieval,

2.2.1 Natural Language Processing for Text Analysis
Natural Language Processing (NLP) encompasses computational techniques for analyz-
ing, understanding, and generating human language. In the context of resume screening,
NLP is applied at multiple stages:
•Tokenization:Splitting text into individual words or subword units for processing.
•Part-of-Speech (POS) Tagging:Identifying grammatical categories (nouns,
verbs, adjectives) to understand sentence structure.
•Named Entity Recognition (NER):Extracting structured entities such as orga-
nization names, dates, locations, and skill keywords from unstructured resume
text.
5\n•Text Preprocessing:Lowercasing, stopword removal, lemmatization, and noise
filtering to prepare text for downstream analysis.
Libraries such as spaCy provide efficient implementations of these NLP pipelines
optimized for production use [11].

7\n2.2.6 Rule-Based Feature Extraction for Validation
The validation module extracts structured features from parsed resumes to identify
inconsistencies and suspicious patterns. Key validation features include:
•Experience Timeline Consistency:Detection of overlapping employment dates,
unrealistic promotion timelines, and graduation year versus claimed experience
mismatches.
•Skill–Experience Alignment:Verification that claimed skill levels are supported
by corresponding project work and employment history.
•Achievement Density:Measurement of quantitative achievements (percentages,
metrics, numerical results) per resume section, as authentic resumes typically
contain measurable outcomes.
•Generic Phrase Frequency:Detection of overused, non-specific language pat-
terns commonly associated with AI-generated content.
•Employment Gap Analysis:Identification of unexplained gaps in the employ-
ment timeline.
These extracted features form the input vector for the Decision Tree classifier.

scores. It also shows the risk levels (Low, Medium, or High), explains any

Figure 3.2: Detailed operational flowchart of the proposed system
14\nThe proposed system design, as illustrated in the flowchart above, outlines the sequential
and parallel operations involved in processing candidate resumes against a given job
description. The workflow initiates when a recruiter uploads a candidate’s resume
(in PDF or DOC format) and provides the corresponding job description. The system

specific skills, educational history, and past work experience.

fails due to an unsupported or corrupted file, an error is returned, halting the process.

theall-MiniLM-L6-v2Sentence-BERT model generates dense semantic embeddings

semantic similarity using cosine distance, calculates the skill overlap percentage, and
assesses experience relevance. These metrics are mathematically combined to generate
a customized, weighted match score.
Simultaneously, the validation track focuses on the structural consistency and truth-
fulness of the resume. Utilizing the previously extracted entities, the system derives
validation features such as timeline gaps, achievement density, and the presence of
generic, AI-like phrasing. This feature vector is fed into a trained Decision Tree clas-
sification algorithm, which categorizes the resume as either genuine or suspicious,
ultimately assigning it a low, medium, or high risk level depending on the model’s
confidence. Finally, both tracks converge to produce the ultimate output. Candidates
are ranked based on their weighted match scores and annotated with their respective
validation risk badges. These results are displayed on an interactive recruiter dashboard,
offering a comprehensive view of match percentages, skill gaps, and detailed candidate
analysis to facilitate informed hiring decisions.

3.5 Algorithm
The proposed system follows a structured multi-stage pipeline for AI-based resume
screening and validation. The complete algorithm consists of text extraction, NLP
preprocessing, embedding generation, custom scoring, feature extraction, and classifi-
cation.
3.5.1 Text Extraction and NLP Preprocessing
Resume documents (PDF/DOC) are parsed to extract raw text content. The extracted
text is processed through the spaCy NLP pipeline for:
•Tokenization into words and sentences
•Named Entity Recognition (NER) to extract skills, organizations, dates, and edu-
cational qualifications
•Part-of-Speech (POS) tagging for structural analysis
•Text cleaning including stopword removal and lemmatization
The output is a structured representation of the resume containing separate fields for
skills, experience entries, education records, and free-text descriptions.

The preprocessed resume text and job description are each encoded into fixed-dimensional

ing of the input text.

3.6 Accuracy Evaluation
To evaluate the performance of the proposed system, multiple metrics are used:
3.6.1 Screening Module Evaluation
The screening module is evaluated by comparing system-generated candidate rankings
against human-labeled ground truth:
•Accuracy:Proportion of correct match/non-match classifications.
•Precision:How many top-ranked candidates are truly relevant.
•Recall:How many relevant candidates are captured in the rankings.
•F1-Score:Balanced metric combining precision and recall.
Target: 80–90% accuracy on the custom evaluation dataset.
18\n3.6.2 Validation Module Evaluation
The validation module is evaluated using standard binary classification metrics:
•Confusion Matrix:True positives, true negatives, false positives, and false
negatives.
•Classification Accuracy:Overall proportion of correct genuine/suspicious clas-
sifications.
•Precision and Recall:For both genuine and suspicious classes.
Target: 80–90% classification accuracy with balanced precision and recall.
